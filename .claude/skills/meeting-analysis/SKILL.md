# 会议分析 Skill

将会议笔记/纪要草稿转化为结构化的分析文档，为后续需求分析提供输入。

## 触发场景

当用户提到以下关键词时触发：
- 会议分析、分析会议记录
- 会议整理、会议纪要生成
- 整理会议笔记、提取会议要点
- 会议需求提取
- 用户直接输入会议笔记并要求整理分析

## 工作原则

1. **完整性优先**：不遗漏会议中的关键信息，尤其是决策项和行动项
2. **结构化输出**：严格按照模板的三段式结构输出
3. **标注不确定内容**：对会议中未明确的内容标注为"待确认"
4. **需求关联**：将会议讨论与现有系统功能建立关联
5. **不自行假设**：会议中未提及的内容不得自行补充，必须标注待确认
6. **优先级标注**：所有提取的需求必须标注优先级（P0/P1/P2）

## 输出规范

输出文档包含三个部分：

### 第一部分：结构化会议纪要
- 会议基本信息（时间、地点、参会人员等）
- 议题列表（议题、讨论人、核心观点）
- 决策事项（决策内容、依据、反对意见）
- 行动项（事项、负责人、截止时间、优先级）

### 第二部分：议题分析报告
- 按议题分类整理
- 每个议题包含：背景、讨论要点、分歧点、共识、遗留问题
- 关联的产品模块/功能

### 第三部分：产品需求提取
- 从会议中提取的需求清单
- 每条需求包含：描述、提出人、优先级、关联议题、初步评估
- 待确认事项列表

## 输出目录

文档保存在 `prd/会议分析/` 目录下，命名格式：`YYYY-MM-DD_会议主题.md`

## 工作流集成

本 skill 是需求分析的前置环节：

```
会议分析 → 需求分析 → PRD 撰写
```

- 会议分析输出作为需求分析的输入材料
- 需求分析阶段需引用会议分析文档
- PRD 撰写需基于需求分析文档（间接引用会议内容）

## 使用说明

1. 用户提供会议笔记/纪要草稿
2. 按照模板进行结构化分析
3. 输出文档保存到 `prd/会议分析/` 目录
4. 如有重要遗漏或不确定内容，主动与用户确认
