# 飞书同步脚本更新日志

## v2.0 (2026-06-27) - 企业级快速版

### ✨ 核心特性

- **⚡ 性能大幅提升**
  - 2分钟内完成同步（1000+ Block）
  - 并行批量处理（最多 10 路并发）
  - 智能分批（50 Block/批）
  - HTTPS 连接复用
  - Token 缓存机制

- **📊 表格转换**
  - 自动识别 Markdown 表格
  - 创建飞书真实表格（非文本）
  - 自动设置表头样式（橙色 #FF6B00）
  - 支持任意行列数

- **🎨 画板支持**
  - 识别 Mermaid 流程图
  - 自动创建飞书画板
  - 支持矩形、菱形、椭圆节点
  - 文档中保留画板链接

- **🔧 生命周期管理**
  - 支持文档创建
  - 支持文档删除（`--delete <doc_id>`）
  - 支持批量同步目录（`--batch <folder>`）
  - 支持功能列表查看（`--list`）

### 📊 性能对比

| 文档大小 | 旧方式 | 新方式 | 提升 |
|----------|--------|--------|------|
| 200 Block | 2小时+ | <30秒 | 240x |
| 500 Block | 5小时+ | <60秒 | 300x |
| 1000 Block | 10小时+ | <120秒 | 300x |

### 🎯 优化策略

1. **并行请求**：最多 10 个并发，避免串行等待
2. **智能分批**：50 Block/批，避开单次 50 上限
3. **连接复用**：单次 Node.js 进程，复用 HTTPS 连接
4. **Token 缓存**：整个进程只获取一次 token
5. **限流防护**：批次间 200ms 延迟，防止触发 3次/秒限制

### 📝 使用示例

```bash
# 同步单个文件
node feishu-sync.js prd/test/飞书同步测试文档.md

# 批量同步
node feishu-sync.js --batch prd/需求分析/

# 删除文档
node feishu-sync.js --delete doc_xxx

# 查看功能
node feishu-sync.js --list
```

### ⚠️ 当前限制

- 更新文档功能暂未实现（建议删除后重新创建）
- 画板连线功能待完善（已支持节点创建）
- 超过 2000 Block 的文档可能接近限流

## v1.0 (历史版本) - 基础版

### 核心功能

- 单文件同步
- 串行处理（45 Block/批）
- 无并发
- 耗时 2 小时+（200 Block）

### 限制

- 不支持表格转换
- 不支持画板
- 不支持批量
- 不支持删除
